## 一、已完成

### 1. 核心平台（基线）

| 模块 | 能力 |
|------|------|
| 部署 | Docker Compose 全栈、`bundle` 单 Jar、安装脚本（`install.sh` / `install.ps1`） |
| 知识库 | 工作区、PUBLIC/PRIVATE、成员 ACL（READ/WRITE/ADMIN） |
| 文档 | PDF/DOCX/TXT/MD/PPTX/XLSX/HTML 上传、异步索引、分块预览、重建索引 |
| 检索 | PgVector **HYBRID**（向量 + 全文 RRF）、可选 ONNX/Cohere/Jina Rerank |
| AI | 智能检索、RAG 问答、多轮流式对话、AI 写文档（附引用来源） |
| LLM | 多提供商 UI 配置（Ollama/DeepSeek/智谱/百炼等），热刷新 |
| Embedding | 默认 ONNX 本地 `bge-small-zh-v1.5` |
| 安全基线 | JWT、首次安装向导、用户管理、基础写操作审计、`ExternalAccessGuard` |

### 2. P0 — 签单三件套

| 功能 | 说明 |
|------|------|
| **LDAP / OIDC 双通道登录** | `LdapAuthService`、`OidcAuthService`、`FederatedUserService`；`KbUser.auth_source` / `external_id`；登录页 SSO 按钮与 OIDC 回调页 |
| **全链路审计** | LOGIN/LOGOUT/SEARCH/QA/CHAT 等事件；`GET /api/audits` 分页；`GET /api/audits/export` CSV；前端审计中心 |
| **隐私策略加固** | Rerank 纳入 `allow-external=false` 管控；`application-enterprise.yml`；`ProductionConfigValidator` 扩展（弱 MinIO 凭据等） |
| **离线交付与合规** | `scripts/offline/prepare-bundle.sh`、`install-offline.sh`；`docs/compliance/data-residency-checklist.md`；`docs/deployment/offline-install.md` |

### 3. P1 — 制造场景与系统集成

| 功能 | 说明 |
|------|------|
| **文档元数据** | `doc_type` / `doc_no` / 生效·失效日期 / 部门 / tags；上传与编辑 API；检索按 `docType` 过滤；`SearchHitVO` 返回文号与章节 |
| **引用溯源** | chunk `section_title` 入库；`RetrievedContentMapper` / `SearchHitMapper` 回填；`SourceList` 点击跳转预览；预览 `chunkId` 高亮 |
| **服务账号 API Key** | `kb_api_key` 表；`X-API-Key` / `Authorization: ApiKey` 鉴权；管理员创建/吊销；前端 API Key 管理页 |
| **备份与恢复** | `scripts/backup/backup-all.sh`、`restore-all.sh`；`docs/deployment/backup-restore.md` |

### 4. 架构重构（v1.0.0 基座）

| 变更 | 说明 |
|------|------|
| LangChain4j 统一向量层 | 移除自研 `vector/*`，统一 `KbEmbeddingStore` + pgvector |
| 仅 PostgreSQL | 移除 SQLite/sqlite-vec 路径 |
| 仅 MinIO 存储 | 移除 `LocalFileStorageProvider` |
| 仅 Redis 缓存 | 移除 Caffeine 本地缓存实现 |

以下能力已有代码或 UI 雏形，**不阻塞 v1.0.0 生产使用**，计划在 v1.0.x / v2.0.0 继续完善：

| **API Key 知识库范围** | 表字段 `kb_id` 已预留 | 创建时绑定 KB、鉴权时校验 KB 访问范围 |

| **集成测试与文档** | 单元测试通过；部分集成测试已更新 | 全量 `IntegrationTest` 在 PG 环境跑通；`docs/api.md` 与新增接口对齐 |
| **安装向导「部署模式」** | 企业 profile 与校验已有 | Setup 向导 UI 增加纯内网 / 允许外连 LLM 选项（可选） |

### P1.5 — RAG 质量与可靠性增强

> 三项改动均为内部工程优化，API 零影响，有 feature flag 兜底可随时关闭。

| 功能 | 当前状态 | 待完成 |
|------|----------|--------|
| **Query Rewrite（多轮指代消解）** | 无 | `QueryRewriter` 接口 + `LlmQueryRewriter` 实现；`ChatServiceImpl.chatStream` 检索前调用改写；改写失败 fallback 原消息 |
| **检索/RAG 质量埋点** | `MetricsService` 已定义但未完整使用 | `SearchService` 用 `timeSearch` 包裹检索；`RagService`/`ChatService` 补全 `timeRag`/`countLlmCall`；新增命中数分布、零结果计数 |
| **异步任务队列 Pub/Sub** | `@Scheduled` 30s 轮询 | Redis Pub/Sub 事件驱动索引；轮询降级为 5min 兜底；重试加指数退避 + DEAD 状态；`IndexEventPublisher` + `IndexMessageListener` |

### 改动量

共约 **10 个文件，~300 行新增代码**，均不改 API 契约。Feature flag 配置项：

```yaml
knowledge.query.rewrite.enabled: true   # 关闭即回退原始消息
knowledge.index.pubsub.enabled: true    # 关闭即回退轮询
```
