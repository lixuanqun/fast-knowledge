# Fast Knowledge v1.0.0

**分支**：`develop_1.0.0`  
**定位**：面向中小企业的 Java 私有化知识库 — 制度/工艺/设备文档智能检索与问答，全链路可内网闭环、可审计交付。  
**目标**：完成签单三件套（P0）+ 制造场景与系统集成（P1），具备内网试点与 POC 验收能力。

---

## 一、已完成

### 1. 核心平台（基线）

| 模块 | 能力 |
|------|------|
| 部署 | Docker Compose 全栈、`bundle` 单 Jar、安装脚本（`install.sh` / `install.ps1`） |
| 知识库 | 工作区、PUBLIC/PRIVATE、成员 ACL（READ/WRITE/ADMIN） |
| 文档 | PDF/DOCX/TXT/MD/PPTX/XLSX/HTML 上传、异步索引、分块预览、重建索引 |
| 检索 | PgVector **HYBRID**（向量 + 全文 RRF）、可选 ONNX/Cohere/Jina Rerank |
| AI | 智能检索、RAG 问答、多轮流式对话、AI 写文档（附引用来源） |
| LLM | 多提供商 UI 配置（Ollama/DeepSeek/智谱/百炼等），热刷新 |
| Embedding | 默认 ONNX 本地 `bge-small-zh-v1.5` |
| 安全基线 | JWT、首次安装向导、用户管理、基础写操作审计、`ExternalAccessGuard` |

| 变更 | 说明 |
|------|------|

### P1.5 — RAG 质量与可靠性增强

> 三项改动均为内部工程优化，API 零影响，有 feature flag 兜底可随时关闭。分析讨论见 [WeKnora 技术对比](../../memory/weknora-analysis.md)。

| 功能 | 当前状态 | 待完成 |
|------|----------|--------|
| **Query Rewrite（多轮指代消解）** | 无 | `QueryRewriter` 接口 + `LlmQueryRewriter` 实现；`ChatServiceImpl.chatStream` 检索前调用改写；改写失败 fallback 原消息 |
| **检索/RAG 质量埋点** | `MetricsService` 已定义但未完整使用 | `SearchService` 用 `timeSearch` 包裹检索；`RagService`/`ChatService` 补全 `timeRag`/`countLlmCall`；新增命中数分布、零结果计数 |
| **异步任务队列 Pub/Sub** | `@Scheduled` 30s 轮询 | Redis Pub/Sub 事件驱动索引；轮询降级为 5min 兜底；重试加指数退避 + DEAD 状态；`IndexEventPublisher` + `IndexMessageListener` |

### 改动量

共约 **10 个文件，~300 行新增代码**，均不改 API 契约。Feature flag 配置项：

```yaml
knowledge.query.rewrite.enabled: true   # 关闭即回退原始消息
knowledge.index.pubsub.enabled: true    # 关闭即回退轮询
```

## 三、v1.0.0 验收标准（建议）

- [ ] `SPRING_PROFILES_ACTIVE=enterprise` + `LLM_ALLOW_EXTERNAL=false` 全链路无外连
- [ ] LDAP 或 OIDC 至少一种与客户 IdP 联调通过
- [ ] 审计日志可分页查询并 CSV 导出
- [ ] 文档带文号上传 → 按类型检索 → 问答引用可跳转预览高亮
- [ ] API Key 调用检索/问答成功并记入审计
- [ ] 备份脚本执行 + 测试环境恢复演练通过
- [ ] 前端 `npm run build`、后端 `mvn test`（含集成测试）通过
- [ ] Query Rewrite：多轮对话指代被正确解析，首轮无历史时原查询不变，改写失败不阻塞对话
- [ ] 检索埋点：`/actuator/metrics` 可查询 `kb.search.latency` / `kb.rag.latency` / `kb.search.hits` 等指标
- [ ] Pub/Sub 任务队列：文档上传后 5s 内触发索引；Redis 不可用时 5min 轮询兜底；重试 3 次后标记 DEAD
