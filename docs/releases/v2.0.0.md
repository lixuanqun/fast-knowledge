# Fast Knowledge v2.0.0（规划，未开发）

**状态**：暂不推荐启动（2026-07 决策）  
**定位**：在 v1.0.0 签单能力之上，形成 **企业版可售卖边界** 与 **LLM Wiki 完整机制**，面向规模化交付与垂直场景模板。

> v1.0.0 聚焦「能签单、能 POC、能内网闭环」；v2.0.0 聚焦「能运营、能差异化、能卖企业版」。

---

## 一、LLM Wiki 完整机制（P2 核心）

| 功能 | 说明 |
|------|------|
| Wiki 维护 Agent | 增量合并、交叉引用自动维护、矛盾 Lint、变更 log、Error Book |
| 查询路由 | Wiki 导航优先（制度/章节型问法）+ RAG HYBRID 兜底 + 双路召回 |
| Wiki UI | Wiki 树浏览、Markdown 编辑、版本 diff、DRAFT 审核流、溯源到 raw 文档 |
| `index.md` | 知识库总目录，供路由与人工浏览 |

---

## 二、企业版功能边界

| 功能 | 说明 |
|------|------|
| `knowledge.edition=enterprise` | 配置层划分社区版 / 企业版能力 |
| 功能对比表 | 文档与代码一致的社区版 vs 企业版能力清单 |
| License 可选 | 12 个月内可用环境变量 + 文档约定，无需 License Server |

**建议企业版独占（相对社区版）**：LDAP/OIDC、审计导出、API Key、完整离线 bundle、制造场景模板、完整 LLM Wiki 编译+维护+审核。

---

## 三、制造 / 国企垂直场景

| 功能 | 说明 |
|------|------|
| 场景模板 | 工艺知识库、设备维保库、制度汇编库（种子数据 + 推荐分块参数） |
| 场景文档 | `docs/scenarios/manufacturing/` 交付说明 |

## 四、系统集成扩展

| 功能 | 说明 |
|------|------|
| 企微 / 钉钉 Webhook | 群消息 → RAG 问答 → 文本回复（轻量 FAQ，非 IM 平台） |

| CAS | 文档说明通过 OIDC 桥接，不单独实现 |

## 五、可运营与交付增强

| 功能 | 说明 |
|------|------|
| RAG 运营面板 | 检索延迟 P50/P95、索引成功率、热门/零结果查询、抽样问答 |
| Wiki 运营面板 | 编译成功率、Lint 告警、待审核页统计 |
| K8s / Helm | 完整清单或 `helm/fast-knowledge/`、PVC/Secret/Ingress 示例 |
| 审计策略文档 | `docs/compliance/audit-policy.md`（留存策略、字段说明） |

---

## 六、明确不做（v2.0.0 仍保持）

避免与 MaxKB / Dify 正面竞争，防止 scope 膨胀：

- 通用可视化 Agent / 工作流编排（Wiki 维护 Agent 除外，且限定单知识库编译域）
- MCP 工具市场
- 多模态（图/音/视频）
- 网页自动爬取
- 多租户 SaaS 运营台
- 独立知识图谱平台 / 复杂组织树 ABAC
- 用 Wiki **完全替代** RAG（万级 raw 文档仍走 pgvector HYBRID）
