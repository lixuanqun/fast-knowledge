# Fast Knowledge 产品说明

> **Fast Knowledge（快速知识库）** — 面向中小企业的开源私有化知识库：单实例部署、数据本地、开箱即用。

---

## 一、产品定位

Fast Knowledge 是一款**轻量级企业知识库**产品，专为「每家企业独立部署一套、数据不出内网、无需 SaaS 多租户」的场景设计。

| 维度 | 定位 |
|------|------|
| **目标用户** | 广大中小企业、团队、部门级知识管理需求 |
| **部署模式** | 单实例私有化，每企一套，非多租户 SaaS |
| **数据规模** | 千级文档、数十用户（轻量场景） |
| **隐私策略** | Privacy by Default：文档、向量、对话默认留在本地服务器 |
| **复杂度** | 开箱即用，渐进式扩展（开发 hash → Docker 演示 → 生产 ONNX/Ollama） |

### 设计原则

1. **Privacy by Default** — 默认本地 Embedding + 可选本地 LLM；支持 `LLM_ALLOW_EXTERNAL=false` 禁止外连大模型
2. **Single Instance** — 单租户架构，轻量工作区组织知识库，无多租户运维负担
3. **Pluggable Backends** — 向量库、缓存、Embedding 一行配置切换，按需升级
4. **Progressive Complexity** — 开发环境可零依赖启动，生产环境按需接入 ONNX / Ollama / pgvector / Qdrant

### 适用场景

- 企业内部制度、手册、技术文档的集中管理与检索
- 销售/客服团队的 FAQ 与产品资料问答
- 研发团队的接口文档、设计文档知识沉淀
- 对数据出境敏感、需完全内网部署的组织
- 希望低成本试用 RAG，后续平滑升级向量与模型能力的团队

### 不适用场景

- 百万级文档、千人以上并发的大规模企业搜索平台
- 需要复杂多租户、计费、SaaS 运营的平台型产品
- 强依赖云端大模型 API、不愿自建任何 AI 组件的场景（本产品强调可本地化）

## 三、核心功能

### 3.1 用户与权限

| 功能 | 说明 |
|------|------|
| JWT 单点登录 | Bearer Token 认证，支持登出与 Token 黑名单 |
| 首次安装向导 | 默认 `admin / admin123`，首次登录强制修改密码并设置实例名称 |
| 角色体系 | `ADMIN`（管理员）、`USER`（普通用户） |
| 用户管理 | 管理员可创建、编辑、禁用用户，重置密码 |
| 审计日志 | 记录知识库创建等关键操作，概览页可查看近期审计 |

### 3.2 知识库管理

| 功能 | 说明 |
|------|------|
| 工作区 | 轻量工作区（`kb_workspace`）组织知识库，自动为用户创建默认工作区 |
| 知识库 CRUD | 创建、编辑、删除知识库，支持名称与描述 |
| 可见性 | `PRIVATE`（私有）/ `PUBLIC`（全实例可见） |
| 成员 ACL | 知识库级成员权限：`READ` / `WRITE` / `ADMIN` |
| 检索参数 | 每个知识库可独立配置混合检索 `alpha` 与 `topK` |

### 3.3 文档管理

| 功能 | 说明 |
|------|------|
| 多格式上传 | PDF、DOCX、TXT、MD、PPTX、XLSX、HTML（Apache Tika 解析） |
| 文档预览 | 在线预览文档内容 |
| 分块查看 | 查看文档索引后的段落分块与 token 计数 |
| 异步索引 | 上传后后台异步向量化，状态：PENDING → INDEXING → INDEXED / FAILED |
| 单文档重建 | 支持对单个文档触发重新索引 |
| 智能写文档保存 | AI 生成的 Markdown 文档可直接保存入库 |
| 单文件限制 | 最大 50MB |

### 3.4 索引与检索

| 功能 | 说明 |
|------|------|
| 智能分块 | 段落感知分块 + Markdown 标题识别 + jtokkit token 计数 |
| 分块参数 | 默认 512 token/块，50 token 重叠（可配置） |
| 混合检索 | Lucene HNSW 向量检索 + SmartChinese BM25 全文检索，加权融合 |
| 检索缓存 | 相同查询结果缓存 5 分钟（Caffeine 或 Redis） |
| 索引任务管理 | 查看待处理/失败任务，支持重试与整库重建索引 |
| 分布式锁 | 索引任务通过缓存 + 数据库双重锁避免重复执行 |

### 3.5 AI 能力

| 模块 | 能力 | 交互方式 |
|------|------|----------|
| **智能检索** | 语义 + 关键词混合搜索，返回相关段落与来源 | 同步 API |
| **智能问答** | 基于知识库 RAG 单轮问答，附引用来源 | 同步 API |
| **智能对话** | 多轮 RAG 对话，SSE 流式输出，会话持久化 | SSE 流式 |
| **智能写文档** | 按主题/大纲/风格生成 Markdown，可引用知识库 | SSE 流式 |

AI 能力基于 **LangChain4j**，对接任意 OpenAI 兼容 API（默认 Ollama + qwen2.5:7b）。RAG 检索结果作为上下文注入，回答严格基于参考资料，不足时明确说明。

### 3.6 运营概览

- 知识库数量、文档总数、已索引/失败文档统计
- 待处理索引任务数量
- 近期审计日志列表

## 四、可插拔技术特性

### 4.1 向量存储（VectorStore SPI）

| 提供方 | 配置值 | 特点 |
|--------|--------|------|
| **Apache Lucene**（默认） | `lucene` | 零外部依赖，内置 HNSW + BM25，适合 SME 低文档量 |
| **pgvector** | `pgvector` | PostgreSQL + pgvector 扩展，适合已有 PG 基础设施 |
| **Qdrant** | `qdrant` | 独立向量数据库，`docker compose --profile qdrant` 启动 |

### 4.2 Embedding 提供方

| 提供方 | 配置值 | 适用场景 |
|--------|--------|----------|
| **ONNX 本地模型**（默认） | `onnx` | 生产离线，`bge-small-zh-v1.5` 中文语义向量 |
| **Ollama** | `ollama` | Docker 演示，`nomic-embed-text` 等 |
| **Hash 降级** | `hash` | 开发联调，无需外部服务（语义质量有限） |

### 4.3 缓存提供方

| 提供方 | 配置值 | 特点 |
|--------|--------|------|
| **Caffeine**（默认） | `caffeine` | 进程内缓存，**零 Redis 依赖**，适合开发与小规模部署 |
| **Redis** | `redis` | 分布式缓存、Token 黑名单、索引锁，适合生产多实例 |

### 4.4 大模型（LLM）

- 兼容 OpenAI API 协议（Ollama、vLLM、各类云端 API）
- 环境变量：`LLM_BASE_URL`、`LLM_API_KEY`、`LLM_MODEL`
- `LLM_ALLOW_EXTERNAL=false` 可禁止连接外部大模型，满足纯内网合规要求

## 五、技术栈

| 层级 | 技术 |
|------|------|
| 前端 | Vue 3、Vite 6、TypeScript、Element Plus、Pinia |

| 缓存 | Caffeine / Redis 7 |
| API 文档 | SpringDoc OpenAPI（Swagger UI） |

## 六、部署方式

| 模式 | 命令 / 方式 | 说明 |
|------|-------------|------|
| **Docker 全栈**（推荐） | `scripts/install.sh` 或 `docker compose -f docker/docker-compose.full.yml up -d` | 应用 + MySQL + Redis，可选 `--profile ai` 启动 Ollama |
| **本地开发** | `scripts/dev.ps1` | 前端 5174 + 后端 8088，默认 Caffeine 无需 Redis |
| **单 Jar 部署** | `npm run build:jar` | 前后端打包为单一 JAR，同端口 8088 托管静态资源 |
| **Nginx 分离** | `npm run build` | 静态 `dist/` + Nginx 反代 API |

默认访问地址：`http://localhost:8088`  
默认账号：`admin / admin123`（首次登录须完成安装向导）

## 七、API 概览

启动后访问 `/api/swagger-ui.html` 查看完整接口文档。

| 分类 | 主要接口 |
|------|----------|
| 认证 | `POST /api/auth/login`、`POST /api/auth/setup`、`POST /api/auth/logout` |
| 系统 | `GET /api/system/config` |
| 知识库 | `GET/POST/PUT/DELETE /api/kbs`、`/api/kbs/{id}/members` |
| 文档 | `POST /api/kbs/{id}/documents/upload`、预览、分块、重建索引 |
| 检索 | `POST /api/search` |
| 问答 | `POST /api/qa` |
| 对话 | `GET/POST /api/chat/sessions`、`POST /api/chat/messages/stream`（SSE） |
| 写文档 | `POST /api/writer/generate`（SSE）、`POST /api/writer/save` |
| 索引任务 | `GET /api/index-tasks/*`、重试、整库重建 |
| 概览 | `GET /api/dashboard/stats`、`/api/dashboard/audits` |
| 用户 | `GET/POST/PUT/DELETE /api/users`、改密、重置密码 |
